## 1. What MemClip is

MemClip is a **privacy-first clipboard history browser extension** (Manifest V3). It
keeps a searchable history of everything you copy — text, links, emails, code —
and, crucially, it records **where each clip was copied from and where it was
pasted to**. That provenance/context tracking is the product's differentiator
(see §9).

- **100% local.** No account, no servers, no analytics, no network requests. All
  data lives in `chrome.storage.local`.
- **Privacy gating:** skips credit-card numbers (Luhn), SSNs, private keys, and
  well-known API-token formats; skips password fields; off in incognito by
  default; per-site denylist.
- **UI:** a pop-out window (not a fixed dropdown) with All / Pinned / Domains /
  Stats / Settings views, fuzzy search, type filters, multi-select bulk actions,
  pin & drag-reorder, export/import backup, and a neon-green "matrix" theme.

## 3. Architecture

Manifest V3. Three runtime contexts, plus a static marketing site.

### 3.1 `background.js` — the single source of truth for WRITES
- Owns **all** mutations to `chrome.storage.local`. Every write runs through an
  in-memory **serialized queue** (`enqueue` / `writeChain`) so concurrent copies
  from multiple tabs, popup actions, and the scheduled purge can never clobber
  each other's read-modify-write cycles.
- Key functions: `addClip`, `recordPaste`, `deleteClip`, `deleteClips`,
  `clearAll`, `pinClip`, `reorderPinned`, `importData`, `purgeOldClips`.
- `saveState()` enforces `MAX_CLIPS` (1000) and on a quota failure **sheds the
  oldest unpinned clips and retries** (up to `QUOTA_RETRY_LIMIT`); pinned clips
  are never shed. Per-clip byte cap is `MAX_CLIP_BYTES` (256 KB) via `capText`.
- Read-only queries (no queue): `getClips`, `getStats`, `getDomains`,
  `openSource`.
- **Incognito gate (authoritative):** the message router checks
  `sender.tab.incognito` and drops captures from incognito tabs unless the user
  opted in. This can't be spoofed and doesn't depend on the content script's own
  detection.
- Window management for the resizable pop-out (remembers size/position).
- Message router at the bottom handles `ADD_CLIP`, `RECORD_PASTE`, `GET_CLIPS`,
  `GET_STATS`, `GET_DOMAINS`, `DELETE_CLIP`, `DELETE_CLIPS`, `REORDER_PINNED`,
  `CLEAR_ALL`, `IMPORT_DATA`, `PIN_CLIP`, `OPEN_SOURCE`, `PURGE_OLD_CLIPS`, and
  debug-log messages.
- Exposes internals via `module.exports` for the Node test harness (no effect in
  the browser).

### 3.2 `content.js` — capture + privacy gate (NO direct writes)
- Listens for `copy` / `paste` events, extracts text (selection, clipboard
  event, or `navigator.clipboard.readText()` fallback for copy buttons / web
  components), then applies the **privacy gate** (`captureBlockReason`:
  incognito, denylist, password field, sensitive data).
- If allowed, sends `ADD_CLIP` / `RECORD_PASTE` to the background. **It does not
  write to storage itself** — this is what removed the old multi-tab race.
- Loads settings from storage and hot-reloads them via `chrome.storage.onChanged`.
- If the extension context is invalidated (see §7), it shows a
  **"MemClip: refresh this page to capture"** toast instead of silently failing.

### 3.3 `popup.html` / `popup.css` / `popup.js` — the UI
- Renders the history list, detail view, domains, stats, settings.
- All history mutations go through the background via messages; only
  settings/meta/UI-state writes happen directly (low-contention).
- i18n via `chrome.i18n` with English fallback (`_locales/en/messages.json`).
- `escapeHtml()` escapes `&`, `<`, `>`, **and quotes**, so it is safe in HTML
  attribute contexts (e.g. `title="…"`).
- **Citation feature** (newest): `buildCitation(clip, format)` + a "Cite this
  clip" section in the detail view (see §5).

### 3.4 `docs/` — GitHub Pages site
- `index.html` (landing, interactive feature carousel), `privacy.html`,
  `support.html` (FormSubmit-backed ticket form). Served at the case-sensitive
  URL `https://chrismmaldonado.github.io/MemClip/`.

## 5. What changed in the most recent session

Two commits, both pushed to `origin/main`.

### Commit `84c390d` — "Route clipboard writes through background queue; harden escaping and privacy copy"
This implemented the fixes from a critical code review. Prioritized "quick wins"
first, then the bigger refactor.

1. **Write-path unification (the big one).** `content.js` used to write directly
   to `chrome.storage.local`, racing the background's queue across tabs. Now it
   sends `ADD_CLIP`/`RECORD_PASTE` and the background is the only writer.
   Removed the now-dead duplicate logic from `content.js` (`commitState`,
   `enforceClipLimit`, `pruneStatsMap`, `shedOldestUnpinned`, plus divergent
   `makeId`/`detectType` copies).
2. **Authoritative incognito guard** in `background.js` via `sender.tab.incognito`
   (closes the content-side "fail open" gap), + stats-map pruning on add/paste.
3. **`escapeHtml` hardening** in `popup.js` — now escapes quotes, so it's safe in
   attribute contexts (prevents attribute-injection from untrusted clip text /
   source URLs / page titles).
4. **Softened overstated privacy claims** (README, STORE_LISTING, docs/index.html,
   popup hint + locale): "API keys are never stored" → "well-known API-token
   formats" since detection is pattern-based, not exhaustive.
5. **Orphaned-context toast** in `content.js` (see §7).
6. **Fixed case-sensitive GitHub Pages URLs** in STORE_LISTING.md back to
   `/MemClip/` (lowercase `/memclip/` returns 404).
7. **Rewired `test/integration.test.js`** to load the real `background.js` and
   exercise the actual content→message→background→storage→popup path, plus two
   new incognito-guard tests.

### Commit `158bca9` — "Add source-attributed 'Cite this clip' actions to the detail view"
The first differentiation feature (see §9).
- `buildCitation(clip, format)` formats a clip three ways:
  - **markdown** — `> <text>\n\n— [<title>](<url>)`
  - **plain** — `"<text>"\n— <title> (<url>), <YYYY-MM-DD>`
  - **link** — `[<title>](<url>)`
- New "Cite this clip" detail-view section, shown **only when the clip has a
  source**. Buttons copy the chosen format with a "Citation copied" toast.
- Refactored `copyClip` into a reusable `copyText(str, msg)`.
- Browser test asserts the cite buttons render and that the Markdown citation
  embeds the source link.

## 7. Known quirk: "copy stopped working after I reloaded the extension"

This came up repeatedly and is **expected dev behavior, not a bug.**

When you reload/update an **unpacked** extension (e.g. via `opera://extensions`
or `chrome://extensions`), any tab that was already open keeps running the
**old, now-orphaned content script** whose extension context is dead. It cannot
capture until that page is refreshed.

- **Fix:** refresh the tab (F5) or open a new tab, then copy.
- The content script now shows a **"MemClip: refresh this page to capture"**
  toast on an orphaned tab so this no longer looks like a silent break.
- End users installing from a store never hit this (single install, fresh tabs).
- It was verified that messaging is reliable even after the service worker fully
  terminates (idle 35s) — a cold copy wakes the SW and is still captured. So the
  message-based write path did **not** regress reliability.

## 9. Product strategy notes (context for decisions)

The clipboard-manager category is crowded and low willingness-to-pay, and a
browser-only tool competes with OS-level managers (Win+V, macOS, Paste, Ditto)
that work everywhere. **The realistic edge is provenance:** MemClip already
stores where each clip came from and where it was pasted. The strategy is to
**lean into a niche that cares about source attribution — researchers, writers,
students** — rather than being a generic clipboard tool.

- The **citation feature** (§5) is the first concrete step on that wedge.
- **Avoid:** novelty/gimmick features (they don't drive retention) and
  **cross-device sync** (high infra cost and it would break the "100% local, no
  servers" privacy promise that is core to the positioning).

## 10. Conventions & gotchas for contributors

- **Shell is PowerShell on Windows.** `&&` and heredocs don't work. For commits,
  write the message to a temp file and use `git commit -F <file>`.
- **Git author identity** is not configured globally. Commit with:
  `git -c user.name="Chrismmaldonado" -c user.email="getmemclip@gmail.com" commit -F <file>`
- **GitHub Pages URLs are case-sensitive** — always use `/MemClip/` (capital M).
- **`docs/` detail-view strings are intentionally hardcoded English**, not i18n;
  the citation UI follows that existing convention.
- Run `npm test` (fast) for logic and `npm run test:browser` (headed Chromium)
  before shipping anything that touches capture, messaging, or the popup.
- After editing the extension during dev, **reload the test tab** before copying.
